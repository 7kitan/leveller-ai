## 2. THE PROBLEM: KEYWORD NOISE ON BOTH SIDES

### Candidate side

- ~28% of IT professionals are career switchers from Marketing, Sales, Finance.
- They manually parse JDs listing 25+ skills, not knowing which 5-6 are actually required.
- They spend 6-12 months "preparing" when they only need 2-3.

Result: missed opportunities, crushed motivation, untapped potential.

### Employer side

Your ATS is rejecting good candidates.

| Classic ATS | What actually happens |
|---|---|
| JD lists 25 keywords → filters by exact match | 80% of viable candidates never reach a human |
| "React" required → CV has "Vue.js" → rejected | False negative. The candidate is 2 weeks from ready. |
| 200+ applicants → recruiter skims top 10 | Random. Based on keyword density, not actual ability. |

**The root cause:** JDs are keyword spam because no one knows which skills *actually* predict success.

### Core capability: Skill proximity, not keyword bingo

ATS models skills as binary checkboxes. Reality is a graph.

> ATS sees: JD needs "React" → CV has "Vue.js" → rejected.
>
> We see: Vue.js developer knows components, state management, hooks, virtual DOM. That's ~80% of React already. **1-2 weeks to ready.**

**Three layers of transferability:**

1. **Same-category transfer** — Vue.js ↔ React, Python ↔ JavaScript. Core concepts carry over; syntax is the only delta.
2. **Adjacent-domain transfer** — Excel → SQL. Different surface, same foundation: data manipulation, logical operations, structured thinking.
3. **Professional strength leverage** — A Sales person switching to a customer-facing tech role brings communication, objection handling, relationship management. These transfer directly.

**Result:** We don't ask "do you have this keyword?" We ask "what do you already know that's close?" Then we calculate the shortest path to readiness.

### Technical stack (brief)

- **Multi-modal CV Parsing** — OCR + LangGraph for any format (PDF, photo, scan)
- **Auto Skill Level Inference** — AI judges proficiency from work context, not self-reports
- **Vector Semantic Search** — pgvector, 1536-dim embeddings, skill proximity matching
- **Real-time Market Crawling** — live data from TopCV and job listings
- **Multi-Platform Course Recommendations** — Coursera, YouTube, and beyond
- **Match Score & Market Demand** — data-driven, market-validated
